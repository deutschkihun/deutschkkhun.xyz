PWA & WebView 에 대한 기초 지식 포스팅

## 1.PWA (Progressive Web App)

HTML, CSS, 자바스크립트(JavaScript) 로 만든 웹앱을 모던한 웹 브라우저 API와 결합하여 크로스 플랫폼에서 동작하는 어플리케이션을 말한다.

기존의 ios 나 android 어플리케이션은 각각의 플랫폼에서 제공하는 api 를 이용 해서 개발 할 수 있지만, 앱스토어,플레이스토어 에서만 사용이 가능하다는 제약 사항이 존재한다. 반대로 브라우저에서 동작하는 웹앱은 플랫폼에 상관없이 브라우저만 있으면 사용가능하므로 사람들이 쉽게 접근해서 사용 할 수 있는 장점이 있다. 하지만 네이티브 앱처럼 플랫폼 자체의 api 를 사용 할 수 없다는 단점이 있다.

PWA 는 접근성이 높은 웹앱의 장점과 플랫폼의 api 를 사용 할 수 있는 네이티브 앱의 장점의 결합되어 있다고 이해하면 편하다.

- 네이티브: 쉽게 말해 android,ios 를 말한다
- 크로스폴랫폼: 하나의 소프트웨어가 서로다은 풀랫폼에서 지원가능 한 것을 말한다. 대표적 크로스 플랫폼 framework 에는 react-native,flutter 를 말한다. 동일한 코드로 여러 플랫폼에서 작동 할 수 있게 하니 android,ios 를 따로 만들 필요가 없고 하나만 만들면 둘다 작동 가능함. 이렇게 만든 크로스 플랫폼에서 동작하는 앱을 **하이브리드 앱**이라고 한다.

- 하이브리드 앱 vs PWA: 둘은 비슷한 것 같지만 엄밀히 말하자면 차이가 있다. PWA 는 웹 브라우저기반에서 시작하여 앱처럼 동작하는 웹앱이지만, 하이브리드 앱 은 android,ios 같은 네이티브 앱에 플랫폼 상관없이 모두 실행 할 수 있게 만들어주는 모바일기반의 앱이다.

## 2. PWA 를 만들기 위해 고려해야 하는 것들

PWA 를 구성하기 위해서는 여러 가지 사항들을 고려해야 하지만 아래 조건들은 거의 필수적인 요소들이다.

- application manifest
- service worker
- push notification
- responsive web design
- native-app like
- installable on user's device

## 3. PWA 제약사항

PWA 가 좋은 부분도 많이 있지만 제약사항도 많이 있다. 우선 특정 브라우저 혹은 이전 브라우저 버전에서 동작하지 않거나 지원하지 않는 경우가 있다. 예를 들면 파이어폭스. 그리고 네이티브 앱처럼 하지만 네이티브 앱처럼 플랫폼 자체의 api 를 자유롭게 사용 할 수 없다는 일부 단점이 있어, 완전히 네이티브 앱처럼 동작하게끔 구현하는데에는 한계가 있습니다.

- javascript,css,html 기반으로 만든 웹 코드 및 웹 어플리케이션
- https
- aplication manifest.json
- service-worker.js

## 5. WebView

- WebView는 App에서 웹브라우저를 이용해 화면을 보여주는 방식

- 예를 들어, 카카오톡에서 대화상대와 링크를 주고 받으면 카카오톡 앱에서 웹뷰로 해당링크에 접속

## 6. WebView 를 사용하는 이유 그리고 문제점

WebView 를 사용하는 이유

- 웹뷰(Webview)를 이용하여 개발하면, 웹과 App을 한번에 개발하는 것이기 때문에 개발 속도가 빠르고, 사용자에게 같은 이용 경험을 제공할 수 있다.

- 반응형만 잘 대비해두면, 서비스를 이용할 때 고객은 앱에서는 물론 웹에서도 같은 경험을 할 수 있다.

WebView 의 문제점

- native api 를 사용하기 어려움, 앱 사용간 native 스러운 부드러움이 떨어짐
- 깜빡이는 하얀 화면
